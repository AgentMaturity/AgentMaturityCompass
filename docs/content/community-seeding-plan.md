## Engagement Rules
1. **Never spam** — one post per community, respond to all comments
2. **Lead with value** — share the tool, not marketing language
3. **Be honest about limitations** — builds trust faster than hype
4. **Engage with criticism** — every critique is a feature request
5. **Cross-link sparingly** — HN post links to GitHub, not to every other post

## Content Types per Platform
| Platform | Format | Tone |
|---|---|---|
| HN | Technical, concise, Show HN format | Builder, matter-of-fact |
| Reddit | Longer, personal story, "I built" format | Genuine, community-first |
| dev.to | Tutorial with code blocks | Practical, step-by-step |
| LinkedIn | Problem → solution → results | Professional, outcome-focused |
| Twitter/X | Thread with screenshots | Punchy, visual, shareable |
| ProductHunt | Launch page + description | Polished, feature-focused |
