Section 01

Executive Summary

The client operates a mid-scale AI agent program spanning customer support automation, internal knowledge retrieval, and lead qualification — deployed across three business units with approximately 14 active agent workflows. Our Compass Sprint assessment reveals a program at early-to-mid maturity (weighted average L2.7), with notable strength in evaluation practices but significant gaps in governance, cost management, and cross-team orchestration.

The program has achieved meaningful production deployments and clear business value in its support automation vertical. However, the absence of a centralized governance framework and inconsistent evaluation standards across teams create compounding risk as the organization scales from 14 to a projected 40+ agent workflows over the next two quarters.

This report provides a dimension-by-dimension maturity assessment, prioritized gaps, and a concrete 30/60/90-day roadmap to advance the program from reactive to structured maturity — unlocking the operational leverage needed for the next phase of growth.

Section 03

Dimension Deep-Dive: Evaluation

Evaluation is the client's strongest dimension, reflecting early investment by the ML platform team in automated testing infrastructure.

Current State

  • 9 of 14 production agents have automated evaluation suites running on every pull request
  • Custom rubric-based scoring for support ticket quality (accuracy, tone, resolution completeness)
  • Regression test sets maintained with ~200 golden examples per agent vertical
  • Eval results surfaced in GitHub PR comments with pass/fail gating

Identified Gaps

  • 5 agents (lead qualification, 2× internal knowledge, 2× data extraction) have zero automated evaluation
  • No adversarial or red-team evaluation for any agent — prompt injection and jailbreak resilience untested
  • Eval datasets are static; no mechanism to incorporate production failures into test sets
  • Cross-agent evaluation (e.g., handoff quality between support triage → specialist agent) not measured

Recommendations

R-EVAL-01: Extend eval coverage to all 14 agents within 30 days. Prioritize lead qualification agent given revenue impact.
R-EVAL-02: Implement adversarial eval suite using OWASP LLM Top 10 as baseline. Target: 60-day delivery.
R-EVAL-03: Build production-feedback loop — auto-flag low-confidence or user-escalated interactions for eval set inclusion.
R-EVAL-04: Define cross-agent handoff metrics: context preservation score, latency at boundary, user satisfaction delta.

Section 04

Dimension Deep-Dive: Governance

Governance represents the most critical gap in the client's agent program. Without centralized oversight, the organization faces escalating compliance risk as agent count grows.

Current State

  • No formal agent deployment approval process — teams self-deploy to production
  • System prompts stored inconsistently: some in version control, others in platform UIs
  • No change audit trail — unable to reconstruct what prompt or model version was live on any given date
  • Data access permissions granted at team level, not agent level — over-provisioned by default

Identified Gaps

  • Zero documentation of agent capabilities, boundaries, or authorized actions
  • No incident response playbook for agent failures or harmful outputs
  • Regulatory exposure: customer-facing agents process PII without documented data handling policies
  • No periodic review cadence — agents deployed 8+ months ago have never been re-evaluated

Recommendations

R-GOV-01: Establish Agent Registry as single source of truth for all deployed agents within 14 days.
R-GOV-02: Implement mandatory deployment gates: eval pass, security review, data classification check.
R-GOV-03: Create Agent Cards (model cards adapted for agents) documenting capabilities, limitations, and authorized scope.
R-GOV-04: Define quarterly agent review cadence with mandatory re-evaluation against updated benchmarks.

Section 05

30 / 60 / 90-Day Roadmap

📍 Days 1–30 — Foundation
  • Week 1: Launch Agent Registry — catalog all 14 agents with owner, status, data access, model version
  • Week 2: Implement per-agent cost tagging via LLM gateway; deploy budget alert thresholds
  • Week 3: Extend eval coverage to remaining 5 agents; prioritize lead qualification
  • Week 4: Draft governance policy v1 — deployment gates, change management, incident response template
📍 Days 31–60 — Hardening
  • Week 5–6: Deploy unified observability dashboard; consolidate LangSmith + custom traces
  • Week 6–7: Implement adversarial eval suite; run first red-team exercise across all customer-facing agents
  • Week 7–8: Build confidence-based routing for human-in-the-loop escalation; pilot on support vertical
📍 Days 61–90 — Scale
  • Week 9–10: Deploy multi-agent orchestration framework; migrate linear chains to DAG-based coordination
  • Week 10–11: Launch agent template library — standardized patterns for common agent types
  • Week 11–12: Conduct first quarterly agent review; update maturity scores; set Q2 targets
